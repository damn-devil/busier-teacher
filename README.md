# BSUIR Teacher Schedule Bot

Telegram-бот для преподавателей БГУИР. Отправляет ID преподавателя — и бот присылает расписание и автоматические уведомления о парах с учётом числителя/знаменателя.

## Команды

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и инструкция |
| `/schedule` | Расписание на сегодня (с фильтром по текущей неделе) |
| `/tomorrow` | Расписание на завтра |
| `/week` | Полное расписание на неделю |
| `/current` | Текущая пара + сколько минут до конца |
| `/next` | Следующая пара + через сколько начнётся |
| `/reset` | Сбросить регистрацию и зарегистрировать другого преподавателя |
| `/stats` | Статистика (только для администратора) |
| `/help` | Справка |

## Автоуведомления

Бот приходит сам, без команд, в фоне:

- **🌅 За 30 мин до первой пары** — будильник + полное расписание на день
- **🎯 Начало пары** — предмет, тип, аудитория, группы, длительность
- **☕ Перерыв 5 мин** — через 40 мин после начала пары + сколько осталось
- **⏰ За 5 мин до конца пары** — предупреждение
- **✅ Конец пары** — завершение + информация о следующей паре
- **⏰ За 5 мин до следующей пары** — только если перемена ≥ 10 мин (после длинных перемен)

Все уведомления по минскому времени (UTC+3).

## Как это работает

```
Пользователь → Telegram → Polling (PTB) → API БГУИР → данные в ответ
                                       → Supabase (сохранение)
                                       → Уведомления (scheduler)
```

- **Polling** — бот постоянно получает сообщения от Telegram
- **Flask** — health-check для Render (чтобы платформа не убивала процесс)
- **Scheduler** — отдельный поток, каждые 30 секунд проверяет время и отправляет уведомления
- **Supabase** — REST API через `requests` (без `supabase-py`)

## Фильтрация по неделям

В БГУИР пары делятся на числитель (неделя 1) и знаменатель (неделя 2). Бот автоматически:
- Получает номер текущей недели через `iis.bsuir.by/api/v1/schedule/current-week`
- Показывает только те пары, у которых `weekNumber` совпадает с текущей неделей

## Стек

- **Python 3.14** + `python-telegram-bot 22.x`
- **Supabase** (PostgreSQL) — хранение данных
- **Flask** — health-check
- **Render** — хостинг

## Переменные окружения

| Переменная | Описание |
|------------|----------|
| `BOT_TOKEN` | Токен бота от BotFather |
| `ADMIN_CHAT_ID` | Ваш Telegram chat ID (для `/stats`) |
| `SUPABASE_URL` | Project URL из Supabase |
| `SUPABASE_ANON_KEY` | Anon public key из Supabase |
| `PORT` | Порт (Render задаёт автоматически) |

## Локальный запуск

```bash
cp .env.example .env
pip install -r requirements.txt
python api/bot.py
```

## Деплой на Render

1. Создайте Web Service, подключите репозиторий
2. Start Command: `python api/bot.py`
3. Добавьте переменные окружения (см. выше)
4. Настройте cron-job.org на `https://ваш-сервис.onrender.com/` каждые 5 минут (чтобы Render не засыпал)
